机器学习基本概念

Posted on 2019-01-04

监督学习(Supervised learning):训练数据集含标签

监督式学习（Supervised learning），是一个机器学习中的方法，可以由训练资料中学到或建立一个模式（ learning model），并依此模式推测新的实例。训练资料是由输入物件（通常是向量）和预期输出所组成。函数的输出可以是一个连续的值（称为回归分析），或是预测一个分类标签（称作分类）。
常用算法：线性回归、分类

无监督学习(Unsupervised learning):训练数据集无标签

无监督式学习(Unsupervised Learning )是人工智能网络的一种算法(algorithm)，其目的是去对原始资料进行分类，以便了解资料内部结构。有别于监督式学习网络，无监督式学习网络在学习时并不知道其分类结果是否正确，亦即没有受到监督式增强(告诉它何种学习是正确的)。其特点是仅对此种网络提供输入范例，而它会自动从这些范例中找出其潜在类别规则。当学习完毕并经测试后，也可以将之应用到新的案例上。
常用算法：聚类

泛化能力(generalization):训练越多,可泛化能力越强

学习模型适用于新样本的能力。简单来说就是指学习到的模型对未知数据的预测能力。在实际情况中，我们通常通过测试误差来评价学习方法的泛化能力。如果在不考虑数据量不足的情况下出现模型的泛化能力差，那么其原因基本为对损失函数的优化没有达到全局最优。

过拟合(高方差)欠拟合(高偏差)(特征工程没做好/训练样本数量)

过拟合（高方差）：对训练集拟合很好，但是对测试集不能很好的预测。
导致过拟合的原因：预测模型过于复杂（包含过多的特征项：一阶或者高阶），并且往往训练集样本过少。将训练样本自身的一些特征所占权重大。
解决办法：
1、增加样本数量:根据原因，导致过拟合的因素是由于训练集样本太少，不具备整体性，所以增加样本数量可以提升泛化能力。
2、减少变量（属性）数量:与增加样本数量相对，减少无关或者弱相关变量项，可以人工去除，也可以选择相关的算法进行自动去除。
3、清洗数据:导致过拟合的一个原因也有可能是数据不纯导致的，如果出现了过拟合就需要我们重新清洗数据。
4、正则化:减少无关（或者弱相关）参数项的权重，即减小对应的\thetaθ值。进而降低训练集本身特性的影响。
欠拟合（高偏差）：模型未能很好的学习到特征，预测值与现实值相差过大。对于测试集以及训练集的拟合都不好。
导致欠拟合的原因： 预测函数的学习能力不足，预测模型过于简单（具体到预测算法模型中就是：特征项不够（包括原始数据的一阶特征和高阶特征））。高偏差增加训练集对模型误差的影响不大，需要换模型
解决办法：
1、增加特征项:可以添加额外的属性，也可以组合当前属性，生成高阶属性。进而使学习模型的泛化能力更强。
2、减少正则化参数:正则化的目的是用来防止过拟合的，但是现在模型出现了欠拟合，则需要减少正则化参数。

交叉验证(Cross Validation)Cross-validation wiki)

基本思想： 交叉验证的基本思想是把在某种意义下将原始数据(dataset)进行分组,一部分做为训练集(train set),另一部分做为验证集(validation set or test set),首先用训练集对分类器进行训练,再利用验证集来测试训练得到的模型(model),以此来做为评价分类器的性能指标，用来优化模型。总共有三份数据集，参与模型生成的有训练集(train set)、验证集（validation set）以及不参与模型生成的测试集（test set）
数据集分割原则：
数据集分为训练集与测试集
1、训练集样本足够多，至少为总样本数的50%。
2、均匀采样。
Holdout 验证
常识来说，Holdout 验证并非一种交叉验证，因为数据并没有交叉使用。 随机从最初的样本中选出部分，形成交叉验证数据，而剩余的就当做训练数据。 一般来说，少于原本样本三分之一的数据被选做验证数据。
优点：简单只用划分一次数据，训练一次。
缺点：不能充分利用样本数据,并且由于只采用部分数据进行模型训练导致最后的模型与划分数据集的方式有很大相干性，增加了误差。
K-fold cross-validation
K折交叉验证，初始采样分割成K个子样本，一个单独的子样本被保留作为验证模型的数据，其他K-1个样本用来训练。交叉验证重复K次，每个子样本验证一次，平均K次的结果或者使用其它结合方式，最终得到一个单一估测。这个方法的优势在于，同时重复运用随机产生的子样本进行训练和验证，每次的结果验证一次，10折交叉验证是最常用的。
优点：充分利用样本数据。
缺点：K值的选取，训练计算量大。
留一验证
正如名称所建议， 留一验证（LOOCV）意指只使用原本样本中的一项来当做验证资料， 而剩余的则留下来当做训练资料。 这个步骤一直持续到每个样本都被当做一次验证资料。 事实上，这等同于和K-fold 交叉验证是一样的，其中K为原本样本个数。 在某些情况下是存在有效率的演算法，如使用kernel regression 和Tikhonov regularization。
